## v1.1 — 단기 계획 (1~2개월)

### 사용자 경험 개선

**실시간 피드백 루프**
- 스트리밍 응답 중 "이 방향 맞아요 / 아니에요" 즉각 피드백
- 피드백을 선호도 로그에 즉시 반영

**멀티턴 학습**
- 현재: 단일 응답 선택으로 학습
- 목표: 대화 흐름 전체에서 패턴 학습

**음성 입출력 완성**
- 현재: TTS 기본 구현
- 목표: Whisper STT 연동 + 대화 음성 모드

### 기술 부채 해소

- `prisma migrate` 기반 마이그레이션 전환 (현재 `db push`)
- LangGraph 백엔드 ↔ Next.js 프론트엔드 실제 연결
- 테스트 커버리지 확대 (API route 통합 테스트)

## v2.0 — 중기 계획 (3~6개월)

### 협업형 AI 어시스턴트

```
현재: 1사용자 → 1AI 개인화
목표: 팀 공유 메모리 + 개인 메모리 분리
```

- 팀 워크스페이스: 공유 페르소나, 공유 플로우
- 개인 선호도는 팀과 분리 유지
- 역할별 권한 관리 (Admin / Member / Guest)

### 멀티모달 확장

| 현재 | v2.0 목표 |
|------|-----------|
| 텍스트 + 이미지 입력 | 영상, 음성, 문서 전체 파이프라인 |
| 텍스트 출력 | 코드 실행 환경 (Vercel Sandbox) |
| 웹 검색 | MCP 툴체인 (GitHub, Calendar, Notion) |

### 고급 학습 알고리즘

- **DPO (Direct Preference Optimization)**: 수집된 선호도 로그로 파인튜닝 데이터셋 자동 생성
- **온라인 강화학습**: 실시간 피드백으로 랭킹 모델 업데이트
- **글로벌 학습**: 전체 사용자 패턴에서 공통 인사이트 추출 → 개인 메모리에 반영

## v3.0 — 장기 비전

### "AI가 나를 설계하는" 경험

현재 시스템은 사용자가 선택 → AI가 학습하는 단방향입니다.
v3.0에서는 AI가 먼저 제안하고, 사용자가 승인하는 양방향으로 진화합니다.

```
AI: "지난 3주간 기술 문서를 작성할 때 항상 STRUCTURED 스타일을 
     선택하셨네요. 기술 글쓰기를 기본 스타일로 설정할까요?"

User: "응, 맞아. 설정해줘."

AI: 자동으로 PreferenceMemory 업데이트 + 확인 메시지
```

**자율 에이전트 파이프라인**
- LangGraph 실제 AI 호출 연동
- 복잡한 태스크를 여러 에이전트가 병렬 처리
- 에이전트 간 컨텍스트 공유

## 기술적 회고

### 잘 된 것

**Provider 추상화 패턴**
OpenAI / Anthropic / Google / Mock을 환경변수 하나로 전환하는 구조는 개발 초기부터 설계해 큰 효과를 봤습니다. 실제 API 없이도 전체 UI/UX를 완성할 수 있었습니다.

**Promise.allSettled로 후보 생성**
3개 후보를 병렬로 생성하되, 하나가 실패해도 나머지를 반환합니다. 실패를 허용하는 설계 덕분에 API 오류가 UX 전체를 망가뜨리지 않습니다.

**JSON 직렬화 레이어**
SQLite 호환성을 위해 배열/객체를 문자열로 저장했는데, PostgreSQL로 전환할 때도 코드 변경 없이 동작했습니다.

### 어려웠던 것

**세션 일관성 문제**
NextAuth JWT 세션 + 익명 쿠키 세션을 동시에 지원하면서, 두 세션의 userId를 일관성 있게 resolve하는 것이 복잡했습니다. `resolveUserContext()` 함수로 통합했습니다.

**Vercel 서버리스 환경의 제약**
서버가 Washington DC(UTC)에서 실행되어 한국 시간(KST)을 잘못 반환하는 문제가 있었습니다. `Intl.DateTimeFormat`으로 KST를 명시적으로 변환하여 해결했습니다.

**LLM 응답 일관성**
후보 생성 시 3개의 응답이 명확하게 다른 스타일을 유지해야 하는데, 초기에는 STRUCTURED와 ANALYTICAL이 비슷하게 나오는 문제가 있었습니다. 각 전략에 "CRITICAL RULES"를 추가해 일관성을 강제했습니다.

### 다음에 다르게 할 것

1. **마이그레이션 파일부터** — `db push` 대신 `migrate dev`로 시작해야 프로덕션 배포가 안전합니다.
2. **통합 테스트 먼저** — 단위 테스트는 있지만 API route 통합 테스트가 없어, 프로덕션에서 처음 발견하는 버그들이 있었습니다.
3. **환경 변수 문서화** — `.env.example`을 처음부터 관리했다면 Vercel 배포 시 혼란이 없었을 것입니다.
