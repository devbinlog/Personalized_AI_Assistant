# 01 프로젝트 개요 (Project Overview)

## 한 문장 정의

Personalized AI Assistant는 사용자의 선택을 학습하여, 대화를 이어갈수록 사용자에게 가장 알맞은 AI 경험을 제공하는 개인화 AI 비서입니다.

---

## 왜 만들었는가

기존 AI 챗봇은 매 대화를 "처음 만나는 사람"처럼 시작합니다. 사용자가 "간결하게 답해줘", "단계별로 설명해줘"라고 반복해서 요청해도, 다음 세션에서 AI는 그것을 기억하지 못합니다.

더 근본적인 문제는 **왜 이 답변이 나왔는지 알 수 없다**는 점입니다. ChatGPT 메모리를 켜도 AI가 무엇을 어떻게 기억하는지, 어떤 이유로 이 응답을 선택했는지 사용자에게 보이지 않습니다.

이 프로젝트는 두 가지 문제를 동시에 해결하려 합니다.

1. **학습**: AI가 사용자의 응답 전략 선호를 능동적으로 학습한다
2. **투명성**: 모든 AI 결정에 이유가 명시된다 (Explainable AI)

---

## 핵심 아이디어

```
"단순히 정보를 기억하는 것이 아니라,
 응답 전략 자체가 발전하는 AI를 만든다"
```

### 기존 AI 메모리와의 차이

| 구분 | 기존 AI (예: ChatGPT 메모리) | 이 프로젝트 |
|---|---|---|
| 학습 방식 | AI가 알아서 판단하여 저장 | **사용자가 직접 선택하며 가르친다** |
| 투명성 | 블랙박스 | **모든 결정에 이유가 표시된다** |
| 학습 대상 | 사실 정보(이름, 직업 등) | **응답 전략의 선호 패턴** |
| 데이터 | 서비스 내부 | **사용자 소유 DB** |
| LLM 교체 | 불가 | **환경변수 한 줄** |

---

## 해결하는 문제

### 문제 1: AI는 내 응답 스타일 선호를 학습하지 못한다

사용자마다 좋아하는 응답 스타일이 다릅니다. 어떤 사람은 핵심만 간결하게, 어떤 사람은 단계별 구조화된 설명을, 어떤 사람은 개념부터 차근차근 설명해주길 원합니다.

이 프로젝트의 Learning Mode는 같은 질문에 서로 다른 전략으로 작성된 3개의 후보 응답을 제시합니다. 사용자가 하나를 선택할 때마다 그 선택이 Preference Log (선호 로그)로 저장됩니다. 5개 이상의 로그가 쌓이면 LLM이 패턴을 분석해 Preference Memory (선호 메모리)를 합성합니다. 이 메모리가 이후 모든 응답 생성에 주입됩니다.

### 문제 2: AI가 왜 이 답변을 냈는지 알 수 없다

기존 AI는 응답을 하나만 내보냅니다. 사용자는 그것이 최선인지, 다른 스타일도 가능했는지 알 수 없습니다.

이 프로젝트의 XAI (Explainable AI) 패널은 다음을 투명하게 공개합니다.

- 왜 이 전략이 선택됐는가 (Preference Memory 영향도)
- 다른 후보들은 몇 점을 받았는가 (랭킹 상세)
- 어떤 페르소나가 응답 스타일에 영향을 줬는가
- 전체 신뢰도 점수 (5가지 차원의 가중 평균)

## 주요 기능 목록

### 채팅 코어

| 기능 | 설명 |
|---|---|
| Learning Mode | 3개의 전략별 후보 응답 비교 선택 |
| Normal Mode | Preference Memory 기반 자동 최적 응답 스트리밍 |
| 파일 첨부 | 이미지 (Vision), 텍스트, PDF 코드 읽기 |
| 웹 검색 | Tavily 연동 실시간 검색, 최신 정보 기반 답변 |
| 대화 히스토리 | 계정 기반 대화 저장, 사이드바 탐색 |

### 학습 시스템

| 기능 | 설명 |
|---|---|
| Preference Log | 선택 전략, 이유 태그, 태스크 유형 저장 |
| Preference Memory | LLM이 로그를 분석해 사용자 프로필 합성 |
| Memory Versioning | 메모리 변화 이력 추적 (diff 보존) |
| Adaptive Suggestion | AI가 패턴 감지 후 능동적 선호 업데이트 제안 |
| Global Learning | 전체 사용자 패턴 집계 및 전역 인사이트 |

### AI 설계 도구

| 기능 | 설명 |
|---|---|
| Persona Studio | 5개 기본 + 사용자 정의 페르소나 설계 |
| Flow Designer | 대화 흐름(단계, 조건, 검색 정책) 설계 및 시뮬레이션 |
| Prompt Lab | 프롬프트 버전 인스펙터, A/B 실험 |

### 분석 및 투명성

| 기능 | 설명 |
|---|---|
| XAI Panel | "왜 이 답변인가?" — 전략 선정 이유 공개 |
| Dashboard | 누적 통계, 전략 분포, Memory 버전 히스토리 |
| 18차원 평가 루브릭 | 언어 품질, 사실성, 완성도, 개인화 점수 |
| Dataset Export | DPO 형식 선호도 데이터 내보내기 |

## 프로젝트 범위와 의도

이 프로젝트는 두 가지 역할을 합니다.

**포트폴리오 프로젝트로서**: LLM 오케스트레이션, 멀티에이전트 파이프라인, 개인화 학습 루프, Explainable AI, LLM Provider 추상화를 모두 실제 코드로 구현한 포트폴리오입니다.

**실용 제품으로서**: 실제 배포된 서비스(Vercel)로 사용자가 지금 당장 사용할 수 있습니다. Mock LLM을 기본으로 구조를 먼저 경험할 수 있고, API 키 연결 시 완전한 AI 기능으로 전환됩니다.

| Phase 5-6 | Memory Engine, Prompt Builder |
